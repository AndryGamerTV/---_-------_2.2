Структура системи

Клієнт ──► Бронювання ──► Об'єкт
   │           │             │
   │           ├─ статус ─► Нове / Підтверджено / Заїзд / Виїзд / Скасовано
   │           └─ впливає ─► KPI дашборду
   │
   └─ пов'язаний ─► Фінанси ──► Дохід / Витрата

Об'єкт ── статус ──► Вільний / Зайнятий / На прибиранні / На ремонті
Об'єкт ───► Задача ──► Очікує / В процесі / Виконано
Аналітика ── агрегує ──► Бронювання + Фінанси + Завантаженість

Основні зв'язки

  • Клієнт → Бронювання (один клієнт може мати кілька бронювань)
  • Бронювання → Об'єкт (одне бронювання прив'язане до одного об'єкта)
  • Об'єкт → Задача (одному об'єкту можна призначити кілька задач)
  • Бронювання → KPI (статуси впливають на показники дашборду)
  • Фінанси → KPI (дохід та витрати впливають на показники)
